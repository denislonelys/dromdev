РАЗДЕЛ 3: ARENA.AI — ПОЛНОЕ ОПИСАНИЕ САЙТА И КАК С НИМ РАБОТАТЬ
================================================================================

Arena.ai — это веб-платформа для сравнения AI моделей. На сайте есть:
  - Множество AI моделей (GPT-4o, Claude, Gemini, Llama и др.)
  - 4 режима работы: Text, Images, Video, Coding
  - Чат-интерфейс для отправки сообщений
  - Возможность менять модель прямо в интерфейсе

Сайт работает плохо без VPN из некоторых стран. Для доступа используй
MTProto прокси из файла proxy.txt.

3.1 ПЕРВЫЙ ШАГ — КАК ОТКРЫТЬ ARENA.AI ЧЕРЕЗ ПРОКСИ
────────────────────────────────────────────────────────────────────────────────

При запуске агента нужно:

  1. Прочитать proxy.txt и загрузить список прокси
  2. Запустить MTProto туннель (mtg) на локальном SOCKS5 порту
  3. Открыть Playwright браузер через этот SOCKS5 туннель
  4. Перейти на https://arena.ai
  5. Дождаться полной загрузки страницы
  6. Убедиться что сайт открылся (проверить заголовок)

  Если прокси не работает — попробовать следующий из proxy.txt.
  Если все прокси не работают — попробовать прямое соединение.

  Браузер запускать в headless режиме (без графики) на сервере.
  На локальной машине можно запустить с графикой для отладки.

3.2 СТРУКТУРА ИНТЕРФЕЙСА ARENA.AI
────────────────────────────────────────────────────────────────────────────────

Сайт arena.ai имеет следующие основные элементы интерфейса:

  НАВИГАЦИЯ СВЕРХУ:
    Вкладки для переключения режимов работы:
      [Text]  [Images]  [Video]  [Coding]
    Каждая вкладка — кнопка или ссылка в верхней части страницы.

  СЕЛЕКТОР МОДЕЛЕЙ:
    Находится обычно вверху или в боковой панели.
    Позволяет выбрать какую AI модель использовать.
    Может быть реализован как:
      - Выпадающий список (dropdown/select)
      - Кнопки с названиями моделей
      - Карточки моделей для выбора
      - Кастомный UI элемент

  ПОЛЕ ВВОДА:
    Текстовое поле (textarea) для ввода сообщения.
    Обычно находится внизу страницы или в центре.
    Может иметь placeholder: "Type a message...", "Ask anything..." и т.д.

  КНОПКА ОТПРАВКИ:
    Кнопка рядом с полем ввода.
    Иконка стрелки, самолётика, или текст "Send".
    Также работает нажатие Enter или Ctrl+Enter.

  ОБЛАСТЬ ОТВЕТА:
    Блок где появляется ответ AI модели.
    Ответ приходит постепенно (стриминг).
    Может содержать текст, код, изображения.

3.3 КАК ПЕРЕКЛЮЧАТЬ РЕЖИМЫ (Text / Images / Video / Coding)
────────────────────────────────────────────────────────────────────────────────

Режимы — это основные вкладки сайта. Переключение режима меняет тип задачи.

  АЛГОРИТМ ПЕРЕКЛЮЧЕНИЯ РЕЖИМА:

  Шаг 1: Определить текущий активный режим
    Найти элемент с классом "active", "selected", aria-selected="true"
    или похожим атрибутом среди вкладок навигации.

  Шаг 2: Найти нужную вкладку
    Искать по тексту внутри элемента: "Text", "Images", "Video", "Coding"
    Стратегии поиска (пробовать по очереди):
      - button:has-text("Text")
      - [data-tab="text"]
      - [data-mode="text"]
      - nav a:has-text("Text")
      - [role="tab"]:has-text("Text")
      - .tab:has-text("Text")
      - a[href*="text"]

  Шаг 3: Кликнуть на вкладку
    await tab_element.click()

  Шаг 4: Дождаться обновления страницы
    await page.wait_for_load_state('networkidle')
    или
    await page.wait_for_timeout(1500)

  Шаг 5: Проверить что режим переключился
    Проверить что нужная вкладка стала активной.
    Если не переключилась — повторить попытку.

  РЕЖИМЫ И ЧТО ОНИ ДАЮТ:
    Text    → чат с языковыми моделями, ответы текстом
    Images  → генерация изображений по описанию
    Video   → генерация видео по описанию
    Coding  → специализированный режим для написания кода

3.4 КАК ПЕРЕКЛЮЧАТЬ AI МОДЕЛИ
────────────────────────────────────────────────────────────────────────────────

На arena.ai можно выбирать разные AI модели. Это ключевая функция.

  АЛГОРИТМ СМЕНЫ МОДЕЛИ:

  Шаг 1: Найти элемент выбора модели
    Пробовать следующие селекторы:
      - select[name="model"]
      - .model-selector
      - .model-picker
      - [data-testid="model-select"]
      - button.model-name (кнопка с именем текущей модели)
      - .current-model
      - [aria-label*="model"]
      - dropdown с названием модели

  Шаг 2a: Если это HTML select элемент:
    await page.select_option('select[name="model"]', label=model_name)
    или
    await page.select_option('select[name="model"]', value=model_id)

  Шаг 2b: Если это кастомный dropdown:
    # Кликнуть чтобы открыть список
    await trigger_element.click()
    # Подождать появления списка
    await page.wait_for_selector('.model-list, .dropdown-menu, [role="listbox"]')
    # Найти нужную модель в списке
    model_item = await page.query_selector(f'[role="option"]:has-text("{model_name}")')
    или
    model_item = await page.query_selector(f'li:has-text("{model_name}")')
    # Кликнуть на модель
    await model_item.click()

  Шаг 2c: Если это кнопки/карточки:
    model_btn = await page.query_selector(f'[data-model="{model_id}"]')
    await model_btn.click()

  Шаг 3: Дождаться применения выбора
    await page.wait_for_timeout(500)

  Шаг 4: Проверить что модель изменилась
    current = await page.inner_text('.current-model, .selected-model, .model-name')
    Убедиться что текущая модель соответствует выбранной.

  Шаг 5: Если ничего не нашли — сделать скриншот для диагностики
    await page.screenshot(path='screenshots/model_selector_debug.png')
    Логировать: "Не удалось найти селектор модели, скриншот сохранён"

  ВАЖНО: Список моделей зависит от выбранного режима.
    В режиме Text — языковые модели
    В режиме Images — модели генерации изображений
    В режиме Video — модели генерации видео
    В режиме Coding — модели для кода

  ПОЛУЧИТЬ СПИСОК ВСЕХ ДОСТУПНЫХ МОДЕЛЕЙ:
    models = []
    # Открыть dropdown
    # Найти все элементы списка
    items = await page.query_selector_all('[role="option"], .model-option, .model-item, select option')
    for item in items:
        text = await item.inner_text()
        models.append(text.strip())
    return models

3.5 КАК ОТПРАВЛЯТЬ СООБЩЕНИЯ

  АЛГОРИТМ ОТПРАВКИ СООБЩЕНИЯ:

3.6 КАК ПОЛУЧАТЬ ОТВЕТЫ ОТ МОДЕЛИ
────────────────────────────────────────────────────────────────────────────────

  АЛГОРИТМ ПОЛУЧЕНИЯ ОТВЕТА:

  Шаг 1: Дождаться начала ответа
    Ждать появления индикатора загрузки или первого текста ответа.
    Таймаут: 15 секунд на появление ответа.

    Селекторы для проверки начала ответа:
      - .loading-indicator
      - .thinking-indicator
      - .response-text
      - .ai-message
      - .assistant-message
      - [data-role="assistant"]
      - .model-response
      - .chat-message.assistant

  Шаг 2: Ждать окончания стриминга
    Модели отвечают постепенно (текст появляется по частям).
    Нужно ждать пока текст не перестанет обновляться.

    Алгоритм определения конца стриминга:
      last_text = ""
      stable_count = 0
      max_wait = 120  # секунд максимум ждать

      while stable_count < 4 and elapsed < max_wait:
          current_text = получить_текст_ответа()
          if current_text == last_text and current_text != "":
              stable_count += 1
          else:
              stable_count = 0
              last_text = current_text
          подождать 0.5 секунды

    Можно также слушать AJAX запросы через page.on('response') для
    определения конца стриминга по заголовкам.

  Шаг 3: Извлечь финальный текст ответа
    Селекторы для текста ответа:
      - .response-text
      - .ai-message .content
      - .assistant-message
      - [data-role="assistant"] .content
      - .model-output
      - .message-content

    response_text = await response_element.inner_text()

  Шаг 4: Для режима Images — извлечь изображения
    img_elements = await page.query_selector_all(
        '.generated-image img, .result-image, [data-testid="generated-img"]'
    )
    image_urls = []
    for img in img_elements:
        src = await img.get_attribute('src')
        if src:
            image_urls.append(src)

  Шаг 5: Для режима Coding — извлечь код
    code_blocks = await page.query_selector_all('pre code, .code-block, .highlight')
    code_texts = []
    for block in code_blocks:
        text = await block.inner_text()
        if text:
            code_texts.append(text)

  Шаг 6: Вернуть структурированный результат
    {
      "mode": "text",           ← текущий режим
      "model": "gpt-4o",        ← текущая модель
      "prompt": "...",          ← исходный запрос
      "response": "...",        ← текстовый ответ
      "images": [],             ← список URL изображений (для Images режима)
      "code": [],               ← список блоков кода (для Coding режима)
      "latency_ms": 1240,       ← время ответа в мс
      "timestamp": "..."        ← время запроса
    }

РАЗДЕЛ 4: MTProto ПРОКСИ — КАК ИСПОЛЬЗОВАТЬ
================================================================================

4.1 ФАЙЛ proxy.txt — ФОРМАТ И ЧТЕНИЕ
────────────────────────────────────────────────────────────────────────────────

Файл proxy.txt содержит список MTProto прокси. Каждая строка — один прокси.

  ФОРМАТ СТРОКИ:
    HOST:PORT:SECRET
    или
    socks5://USER:PASS@HOST:PORT    (для SOCKS5 прокси)

  ПРИМЕР proxy.txt:
    # Комментарии начинаются с #
    tg.atomic-vpn.com:443:dd3f087f3f403449a2a9446de22b5bc3d1
    193.39.15.115:443:dd585256032fd8a78a0602ddd90f9c981f
    eu.mt-proxy.org:443:eec6c206c4d429f36d13af11fbd3c35e786d742d70726f78792e6f7267
    socks5://tguser:12345@95.81.99.82:1089

  КАК ЧИТАТЬ ФАЙЛ:
    Открыть proxy.txt построчно.
    Пропускать строки начинающиеся с # (комментарии).
    Пропускать пустые строки.
    Парсить формат: разделить по ":"
      Если строка начинается с "socks5://" — это SOCKS5 прокси.
      Иначе — MTProto прокси с полями HOST:PORT:SECRET.

4.2 КАК ЗАПУСТИТЬ MTProto ТУННЕЛЬ
────────────────────────────────────────────────────────────────────────────────

MTProto прокси нельзя использовать напрямую в браузере.
Нужно превратить его в локальный SOCKS5 через инструмент mtg.

  ИНСТРУМЕНТ: mtg (MTProto proxy client)
  Скачать: https://github.com/9seconds/mtg/releases
  Команда:
    mtg run HOST:PORT SECRET --bind-to 127.0.0.1:SOCKS5_PORT

  ПРИМЕР запуска туннеля:
    mtg run tg.atomic-vpn.com:443 dd3f087f3f403449a2a9446de22b5bc3d1 \
        --bind-to 127.0.0.1:11080

  Теперь на 127.0.0.1:11080 работает SOCKS5 прокси.
  Браузер Playwright подключаем через него.

  ЗАПУСК НЕСКОЛЬКИХ ТУННЕЛЕЙ (для ротации):
    Прокси 1 → 127.0.0.1:11080
    Прокси 2 → 127.0.0.1:11081
    Прокси 3 → 127.0.0.1:11082
    ...и так далее

4.3 ПОДКЛЮЧЕНИЕ БРАУЗЕРА ЧЕРЕЗ ПРОКСИ
────────────────────────────────────────────────────────────────────────────────

  После запуска туннеля — передать прокси в Playwright:

    browser = await playwright.chromium.launch(
        headless=True,
        proxy={
            "server": "socks5://127.0.0.1:11080"
        }
    )

  Если это SOCKS5 прокси из файла (без MTProto туннеля):
    browser = await playwright.chromium.launch(
        headless=True,
        proxy={
            "server": f"socks5://{host}:{port}",
            "username": username,
            "password": password
        }
    )

4.4 ПРОВЕРКА РАБОТОСПОСОБНОСТИ ПРОКСИ
────────────────────────────────────────────────────────────────────────────────

  При запуске — проверить все прокси из proxy.txt:

  Алгоритм проверки одного прокси:
    1. Запустить mtg туннель на свободном локальном порту
    2. Попробовать открыть https://arena.ai через этот туннель
    3. Замерить время ответа (latency)
    4. Если статус 200 и время < 10 сек → прокси рабочий
    5. Если ошибка или таймаут → прокси мёртвый
    6. Остановить туннель

  Выбрать прокси с наименьшей задержкой как основной.
  Остальные рабочие прокси — резервные.

  Вывод результатов:
    ✅ tg.atomic-vpn.com:443       — активен (45мс)
    ✅ 193.39.15.115:443            — активен (78мс)
    ❌ free.vpn-direct.com:443      — недоступен
    ✅ eu.mt-proxy.org:443          — активен (112мс)
    Выбран основной: tg.atomic-vpn.com:443 (45мс)

4.5 АВТОМАТИЧЕСКОЕ ПЕРЕКЛЮЧЕНИЕ ПРОКСИ
────────────────────────────────────────────────────────────────────────────────

  Если текущий прокси перестал работать (ошибка соединения):
    1. Логировать: "Прокси X недоступен, переключаюсь..."
    2. Остановить текущий туннель
    3. Взять следующий рабочий прокси из списка
    4. Запустить новый туннель
    5. Пересоздать браузер с новым прокси
    6. Перейти заново на arena.ai
    7. Повторить исходный запрос

  После 3 неудачных попыток всех прокси:
    Попробовать прямое соединение без прокси.
    Если и оно не работает — уведомить пользователя.

  Ротация по количеству запросов:
    После каждых 50 запросов через один прокси — менять на следующий.
    Это снижает риск блокировки.

5.1 ЧТО ЗНАЧИТ "ПАРСИТЬ И ТЕСТИРОВАТЬ В РЕАЛЬНОМ ВРЕМЕНИ"

Это НЕ эмуляция и НЕ прямые API вызовы к моделям.

5.2 ТЕСТ ПРИ ЗАПУСКЕ — САМОПРОВЕРКА

    Результат: PASS ✅ / FAIL ❌ (с указанием какой режим не работает)

  ТЕСТ 3 — Список моделей:
    В каждом режиме открыть список доступных моделей
    Убедиться что список не пустой
    Сохранить список для последующего использования
    Результат: найдено X моделей

  ТЕСТ 4 — Переключение модели:
    Выбрать первую модель из списка
    Убедиться что выбор применился
    Результат: PASS ✅ / FAIL ❌

  ТЕСТ 5 — Отправка тестового сообщения:
    Написать "Hello" в поле ввода
    Отправить
    Дождаться ответа (таймаут 60 сек)
    Убедиться что ответ не пустой
    Результат: PASS ✅ / FAIL ❌

5.3 АДАПТАЦИЯ К ИЗМЕНЕНИЯМ ИНТЕРФЕЙСА
────────────────────────────────────────────────────────────────────────────────

Интерфейс arena.ai может меняться. Агент должен адаптироваться:

  ЕСЛИ CSS селектор не найден — пробовать альтернативы:
    1. Поиск по тексту: has-text("Send"), has-text("Text")
    2. Поиск по роли: [role="button"], [role="tab"], [role="textbox"]
    3. Поиск по placeholder: [placeholder*="message"], [placeholder*="type"]
    4. XPath: //button[contains(text(), "Send")]
    5. По позиции: первая кнопка справа от textarea

  ЕСЛИ интерфейс сильно изменился:
    Сделать скриншот всей страницы
    Проанализировать HTML структуру (page.content())
    Попробовать найти элементы по новым признакам
    Логировать изменения: "Интерфейс изменился, обновляю селекторы..."

  СТРАТЕГИЯ "ПОПРОБУЙ ВСЁ":
    Для каждого действия иметь список из 5-10 способов его выполнить.
    Пробовать их по очереди пока что-то не сработает.
    Если ничего не сработало — логировать подробно и уведомить пользователя.

5.4 ПАРАЛЛЕЛЬНАЯ РАБОТА С НЕСКОЛЬКИМИ МОДЕЛЯМИ
────────────────────────────────────────────────────────────────────────────────

Агент может отправить один запрос в несколько моделей одновременно:

  Алгоритм:
    1. Открыть несколько вкладок браузера (или контекстов)
    2. В каждой вкладке выбрать разную модель
    3. Одновременно отправить одинаковое сообщение во все вкладки
    4. Параллельно ждать ответы (asyncio.gather)
    5. Собрать все ответы
    6. Вернуть сравнительную таблицу

  Ограничения:
    Максимум 5 параллельных запросов (чтобы не перегружать прокси)
    Общий таймаут: 120 секунд

10.1 ЗАПУСК PLAYWRIGHT CHROMIUM
────────────────────────────────────────────────────────────────────────────────

Для парсинга arena.ai используется Playwright с браузером Chromium.
Браузер должен выглядеть как обычный пользователь, а не бот.

  ОБЯЗАТЕЛЬНЫЕ НАСТРОЙКИ ПРИ ЗАПУСКЕ:

  Аргументы запуска Chromium:
    --no-sandbox
    --disable-setuid-sandbox
    --disable-dev-shm-usage
    --disable-blink-features=AutomationControlled
    --disable-infobars
    --disable-extensions
    --no-first-run
    --disable-default-apps
    --disable-gpu  (на серверах без GPU)

  User-Agent (реалистичный, не раскрывающий автоматизацию):
    Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
    (KHTML, like Gecko) Chrome/122.0.0.0 Safari/537.36

  Viewport (размер окна браузера):
    Ширина: 1920 пикселей
    Высота: 1080 пикселей

  Локаль и таймзона:
    Locale: en-US
    Timezone: Europe/Amsterdam  (сервер в Амстердаме)

  Дополнительные HTTP заголовки:
    Accept-Language: en-US,en;q=0.9
    Accept-Encoding: gzip, deflate, br

10.2 СКРЫТИЕ АВТОМАТИЗАЦИИ (STEALTH)
────────────────────────────────────────────────────────────────────────────────

Сайты могут определять что браузер управляется автоматически.
Нужно скрыть признаки автоматизации:

  JavaScript который нужно выполнить при открытии каждой страницы
  (через page.add_init_script):

    // Убрать флаг webdriver
    Object.defineProperty(navigator, 'webdriver', {
      get: () => undefined
    });

    // Имитировать реальные плагины браузера
    Object.defineProperty(navigator, 'plugins', {
      get: () => [1, 2, 3, 4, 5]
    });

    // Имитировать реальные языки
    Object.defineProperty(navigator, 'languages', {
      get: () => ['en-US', 'en', 'ru']
    });

    // Имитировать Chrome runtime (обычно отсутствует в автоматических браузерах)
    window.chrome = {
      runtime: {},
      loadTimes: function() {},
      csi: function() {}
    };

    // Убрать специфические для puppeteer/playwright свойства
    delete window.__playwright;
    delete window.__pw_manual;

    // Имитировать правильное разрешение экрана
    Object.defineProperty(screen, 'width', { get: () => 1920 });
    Object.defineProperty(screen, 'height', { get: () => 1080 });

10.4 ОБХОД CAPTCHA
────────────────────────────────────────────────────────────────────────────────

  Стратегии при появлении CAPTCHA:

  Стратегия 1 — Смена прокси:
    CAPTCHA часто привязана к IP адресу.
    Смена прокси = смена IP = возможно CAPTCHA исчезнет.

  Стратегия 2 — Ждать и попробовать позже:
    Через 5-10 минут CAPTCHA может сняться автоматически.

  Стратегия 3 — 2captcha API (если настроен):
    Если в .env указан TWOCAPTCHA_API_KEY:
      Отправить CAPTCHA изображение в 2captcha сервис
      Получить решение
      Ввести решение на странице

  Стратегия 4 — Уведомить пользователя:
    Если ничего не помогло — сообщить пользователю и попросить
    решить CAPTCHA вручную (в режиме с GUI).

  Команды для установки на чистый Ubuntu 22.04:

  # 1. Обновить систему
  apt update && apt upgrade -y

  # 2. Установить Python 3.11 и зависимости
  apt install -y python3.11 python3.11-pip python3.11-venv
  apt install -y git curl wget

  # 3. Установить зависимости Chromium для Playwright
  apt install -y chromium-browser \
    fonts-liberation libasound2 libatk-bridge2.0-0 libatk1.0-0 \
    libcups2 libdbus-1-3 libdrm2 libgbm1 libgdk-pixbuf2.0-0 \
    libglib2.0-0 libgtk-3-0 libnspr4 libnss3 libpango-1.0-0 \
    libx11-6 libx11-xcb1 libxcb1 libxcomposite1 libxcursor1 \
    libxdamage1 libxext6 libxfixes3 libxi6 libxrandr2 libxrender1 \
    libxss1 libxtst6 xdg-utils

  # 4. Установить mtg (MTProto proxy client)
  wget https://github.com/9seconds/mtg/releases/download/v2.1.7/mtg-2.1.7-linux-amd64.tar.gz
  tar -xzf mtg-2.1.7-linux-amd64.tar.gz
  mv mtg-2.1.7-linux-amd64/mtg /usr/local/bin/mtg
  chmod +x /usr/local/bin/mtg

  # 5. Клонировать IIStudio
  git clone https://github.com/username/iistudio.git /opt/iistudio
  cd /opt/iistudio

  # 6. Создать виртуальное окружение
  python3.11 -m venv venv
  source venv/bin/activate

  # 7. Установить Python зависимости
  pip install -r requirements.txt
  playwright install chromium

  # 8. Настроить конфигурацию
  cp .env.example .env
  nano .env

  # 9. Добавить прокси
  nano proxy.txt

  # 10. Запустить через pm2
  npm install -g pm2
  pm2 start iistudio.py --interpreter /opt/iistudio/venv/bin/python3 \
    --name iistudio
  pm2 save
  pm2 startup

получения API токена и управления балансом IIStudio.

Агент IIStudio при каждом запросе проверяет токен пользователя через orproject.ru.
Без валидного токена — запросы не выполняются.

15.1 КАК РАБОТАЕТ АВТОРИЗАЦИЯ
────────────────────────────────────────────────────────────────────────────────

  СХЕМА АВТОРИЗАЦИИ:

    Пользователь
        │
        │  1. Регистрируется на orproject.ru (email + пароль)
        │     ИЛИ входит через Google (Gmail)
        │
        │  2. Получает API токен вида: username-dev-XXXXX
        │
        │  3. Добавляет токен в IIStudio:
        │       iis --token denialonelys-dev-87169
        │       или в .env: IISTUDIO_TOKEN=denialonelys-dev-87169
        │
        ▼
    IIStudio Agent
        │
        │  При каждом запросе:
        │    POST https://orproject.ru/api/v1/auth/verify
        │    { "token": "denialonelys-dev-87169" }
        │
        │  Если токен валиден → выполнить запрос к arena.ai
        │  Если токен невалиден → отказать: "❌ Неверный токен"
        │  Если баланс = 0 → отказать: "❌ Пополни баланс на orproject.ru"
        │
        │  После успешного ответа от arena.ai:
        │    POST https://orproject.ru/api/v1/billing/charge
        │    { "token": "...", "amount": 0.29 }
        │    → Снять $0.29 с баланса пользователя
        │
        ▼
    arena.ai → ответ → пользователь

15.6 API ЭНДПОИНТЫ ORPROJECT.RU ДЛЯ IISTUDIO
────────────────────────────────────────────────────────────────────────────────

  Агент IIStudio обращается к этим эндпоинтам:

  POST https://orproject.ru/api/v1/auth/verify
    Проверить токен
    Body: { "token": "denialonelys-dev-87169" }
    Ответ OK:
      {
        "valid": true,
        "user": {
          "email": "denislonelys.business@fmail.com",
          "plan": "admin",
          "balance": null,
          "unlimited": true,
          "requests_today": 42,
          "requests_total": 1337
        }
      }
    Ответ ошибка:
      { "valid": false, "error": "Invalid token" }

  POST https://orproject.ru/api/v1/billing/charge
    Списать $0.29 за запрос
    Body: {
      "token": "denialonelys-dev-87169",
      "amount": 0.29,
      "model": "gpt-4o",
      "mode": "text",
      "prompt_preview": "первые 50 символов запроса"
    }
    Ответ OK:
      {
        "success": true,
        "balance_after": 8.42,
        "charged": 0.29
      }
    Ответ — недостаточно средств:
      {
        "success": false,
        "error": "insufficient_balance",
        "balance": 0.15,
        "required": 0.29
      }
    Ответ — Admin тариф (не списывать):
      {
        "success": true,
        "balance_after": null,
        "charged": 0,
        "unlimited": true
      }

  GET https://orproject.ru/api/v1/balance?token=TOKEN
    Получить текущий баланс
    Ответ:
      {
        "balance": 8.42,
        "unlimited": false,
        "plan": "starter"
      }

15.7 КАК IISTUDIO ПРОВЕРЯЕТ ТОКЕН ПРИ КАЖДОМ ЗАПРОСЕ
────────────────────────────────────────────────────────────────────────────────

  АЛГОРИТМ ПРОВЕРКИ ТОКЕНА:

  1. Прочитать токен из:
     а) Аргумента командной строки: --token TOKEN
     б) Переменной окружения: IISTUDIO_TOKEN
     в) Файла конфигурации: ~/.iistudio/config.json поле "token"
     г) Файла .env в рабочей папке: IISTUDIO_TOKEN=

  2. Если токен не найден:
     Показать: "❌ Токен не указан."
     Показать: "Получи токен на orproject.ru и добавь:"
     Показать: "  iis --token YOUR-TOKEN"
     Остановить выполнение.

  3. Отправить запрос на проверку:
     POST https://orproject.ru/api/v1/auth/verify
     { "token": TOKEN }

  4. Если токен невалиден:
     "❌ Неверный токен. Проверь токен на orproject.ru"
     Остановить выполнение.

  5. Проверить баланс:
     Если plan == "admin" → продолжать (безлимитный)
     Если balance < 0.29 → "❌ Недостаточно средств ($X.XX). Пополни на orproject.ru"
     Если balance >= 0.29 → продолжать

  6. Выполнить запрос к arena.ai

  7. После успешного ответа — списать $0.29:
     POST https://orproject.ru/api/v1/billing/charge

  8. Показать пользователю (если не Admin):
     "💳 Списано $0.29 | Остаток: $X.XX"

  КЭШИРОВАНИЕ ПРОВЕРКИ:
    Не проверять токен при каждом из 100 запросов подряд.
    Кэшировать результат проверки на 5 минут.
    Но баланс проверять каждый раз (он меняется).

РАЗДЕЛ 16: НАСТРОЙКА РЕПОЗИТОРИЯ GITHUB
================================================================================

Репозиторий IIStudio настроен следующим образом на GitHub:

16.1 СОЗДАНИЕ И НАСТРОЙКА РЕПОЗИТОРИЯ
────────────────────────────────────────────────────────────────────────────────

  Репозиторий создан командами:
    cd /путь/к/проекту
    git init
    git add .
    git commit -m "feat: initial IIStudio release v3.0"
    git remote add origin https://github.com/USERNAME/iistudio.git
    git branch -M main
    git push -u origin main

  ЗАЩИТА ВЕТОК:
    Ветка main защищена:
      - Требуется Pull Request для merge
      - Минимум 1 ревью
      - Все тесты должны проходить (CI)

  ВЕТКИ РАЗРАБОТКИ:
    main              → стабильная версия (production)
    develop           → разработка (staging)
    feature/*         → новые функции
    fix/*             → исправления багов
    hotfix/*          → срочные исправления production

16.2 GITHUB ACTIONS (CI/CD)
────────────────────────────────────────────────────────────────────────────────

  Файл .github/workflows/ci.yml — автоматические тесты:
    Запускается при каждом push и pull request.
    Проверяет:
      - Синтаксис Python (flake8, mypy)
      - Запускает pytest тесты
      - Проверяет форматирование (black)

  Файл .github/workflows/deploy.yml — деплой на Amsterdam:
    Запускается только при push в ветку main.
    Действия:
      - SSH на Amsterdam сервер
      - git pull
      - pip install -r requirements.txt
      - pm2 restart iistudio

16.3 GITHUB SECRETS (секреты репозитория)
────────────────────────────────────────────────────────────────────────────────

  В настройках репозитория (Settings → Secrets) добавить:
    AMSTERDAM_HOST      → IP адрес Amsterdam сервера
    AMSTERDAM_USER      → пользователь SSH (обычно root)
    AMSTERDAM_SSH_KEY   → приватный SSH ключ для деплоя
    ORPROJECT_API_KEY   → ключ доступа к orproject.ru API

  Эти секреты используются в GitHub Actions для деплоя.
  Никогда не коммить их в код — только через Secrets!

РАЗДЕЛ 18: ИТОГОВЫЙ АЛГОРИТМ ПОЛНОГО ЦИКЛА ЗАПРОСА
================================================================================

Полный цикл от команды пользователя до получения ответа:

  1. Пользователь вводит: iis "напиши функцию на Python"

  2. IIStudio читает токен из конфига/env

  3. Проверяет токен на orproject.ru:
     → Токен валиден, тариф Admin, безлимитный

  4. Определяет режим и модель:
     → Coding режим, Claude 3.5 Sonnet

  5. Показывает прогресс:
     🔄 Coding | Claude 3.5 Sonnet | Прокси: tg.atomic-vpn.com

  6. Если режим/модель не активны — переключает на arena.ai

  7. Вводит сообщение в поле ввода arena.ai

  8. Отправляет сообщение

  9. Ждёт ответа (стриминг)

  10. Извлекает ответ из DOM страницы

  11. Выводит ответ пользователю (с форматированием)

  12. Списывает $0.29 с баланса (или $0 для Admin)

  13. Показывает: ⏱ 2.1с | 💳 $0.00 (Admin) | 🤖 Claude 3.5 Sonnet

  14. Ждёт следующего запроса
